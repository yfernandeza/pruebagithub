# 🤝 Guía de Contribución

¡Gracias por querer contribuir a este taller! Esta guía te ayudará a entender cómo colaborar.

## Cómo Contribuir

### 1. Fork el Repositorio
```bash
git clone https://github.com/yfernandeza/pruebagithub.git
cd pruebagithub
```

### 2. Crear una Rama para tu Feature
```bash
git checkout -b feature/mi-mejora
```

### 3. Realizar Cambios
- Edita los archivos necesarios
- Sigue los estándares del proyecto

### 4. Hacer un Commit Bien Formateado
```bash
git add .
git commit -m "feat: agregar nueva sección sobre git rebase"
```
# CAMBIO
**Formato de commits:**
```
<tipo>: <descripción breve>

<descripción detallada si es necesario>
```

**Tipos de commits:**
- `feat:` - Nueva funcionalidad
- `fix:` - Corrección de errores
- `docs:` - Cambios en documentación
- `style:` - Cambios de formato
- `refactor:` - Refactorización de código
- `test:` - Agregar o actualizar tests

### 5. Push a tu Fork
```bash
git push origin feature/mi-mejora
```

### 6. Crear un Pull Request
- Ve a GitHub
- Haz clic en "New Pull Request"
- Describe tus cambios claramente

## Estándares del Proyecto

### Documentación
- Usa Markdown para toda la documentación
- Mantén líneas menores a 80 caracteres
- Incluye ejemplos de código

### Commits
- Commits pequeños y enfocados
- Mensajes descriptivos
- Referencia issues cuando sea relevante

### Code Review
- Los PRs serán revisados antes de merge
- Responde a los comentarios de revisión
- Realiza cambios solicitados

## Preguntas o Problemas?

- Crea un Issue describiendo tu pregunta
- Usa etiquetas apropiadas
- Proporciona contexto y ejemplos

## Licencia

Al contribuir, aceptas que tu código sea distribuido bajo licencia MIT.
